# Table Monitor — Система детекции состояния столика на видео

## Описание проекта

Проект позволяет анализировать видео с одним столиком и фиксировать следующие события:

- `table_empty` — столик свободен  
- `table_occupied` — столик занят  
- `approach_to_table` — подход посетителя к столу  

Используется YOLOv8 для детекции людей и OpenCV для работы с видео. События сохраняются в CSV, создаётся видео с визуализацией и текстовый отчёт со статистикой.

---

## Установка и запуск

1. Склонируйте репозиторий:

```bash
git clone <URL_репозитория>
cd <имя_папки_репозитория>
```
2.  Создайте виртуальное окружение (рекомендуется):
```bash
python -m venv venv
```
3.  Активируйте виртуальное окружение:
Windows (cmd):
```
venv\Scripts\activate
```
Linux / macOS:
```
source venv/bin/activate
```
4.  Установите зависимости:
```
pip install -r requirements.txt
```
5.  Положите ваше видео в корень репозитория, например video1.mp4.
6.  Запустите скрипт:
```
python main.py --video video1.mp4
```
Выбор зоны столика

При запуске скрипт предложит два способа задания зоны столика:

Прямоугольник через cv2.selectROI — визуально выбираете прямоугольник на первом кадре.
Произвольный полигон — мышкой отмечаете точки, формируя многоугольник. (не менее 3 х точек но не более 13)

Также можно заранее задать координаты в tables_config.json (предварительно создав данный файл.опционально) для автоматической загрузки зоны.

Важно: зона может быть любой формы, важно, чтобы она корректно охватывала столик.

Логика детекции
YOLOv8 детектирует людей на кадре.
Центр каждого bbox проверяется на попадание в полигон столика.
Для уменьшения дрожания используется буфер последних кадров (required_stable_frames=20).
Добавлена задержка empty_timeout_sec=2.0 сек для избежания резкой смены состояния при кратковременной потере детекции.
События фиксируются только после подтверждения стабильного состояния.

Возможные проблемы:

"Дрожание" детекции — кратковременное исчезновение человека может привести к ложным событиям.
Неправильная детекция при плохом освещении, боковом положении человека или частичной перекрытой видимости.

Результаты
Видео с визуализацией: output_(ИМЯ_ЗАПУСКАЕМОГО_ФАЙЛА).mp4 
CSV-файл с событиями: events_(ИМЯ_ЗАПУСКАЕМОГО_ФАЙЛА).csv
Текстовый отчёт: report_(ИМЯ_ЗАПУСКАЕМОГО_ФАЙЛА).txt

Среднее время обработки
Примерное время обработки видео в 15 минут: ~60 минут на обычном ноутбуке с GPU.
Можно ускорить с использованием более мощной видеокарты или уменьшением разрешения.

Примечания
Запускать лучше через CMD или PowerShell на Windows.
Любое видео формата .mp4 подходит, главное, чтобы оно было в корне проекта.
